# Patterns

Named, reusable architectural patterns for enterprise AI systems. Each pattern addresses a recurring problem with a proven solution, documented with when-to-use criteria, trade-offs, and tool guidance.

Patterns are organised by the Control Tower layer they primarily address, though most patterns span multiple layers.

## Pattern Index

| Pattern | Layer | Use When |
|---|---|---|
| [Centralised AI Gateway](./centralised-ai-gateway.md) | 1 | Any enterprise with multiple teams using AI |
| [Hybrid RAG](./hybrid-rag.md) | 2 | Production enterprise knowledge retrieval |
| [Human-in-the-Loop](./human-in-the-loop.md) | 3 | High-stakes AI-assisted decisions |
| [Continuous Evaluation](./continuous-evaluation.md) | 4 | Any AI system in production |
| [AI Workflow Orchestration Bridge](./ai-workflow-orchestration.md) | 5 | Connecting AI to existing enterprise workflows |
