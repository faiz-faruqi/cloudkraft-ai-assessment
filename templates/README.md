# Templates

Engagement templates and deliverable frameworks used in CloudKraft Tier 1, Tier 2, and Tier 3 advisory engagements. Each template is a practitioner-ready starting point — not a blank form, but a structured document with the questions, sections, and decision frameworks that drive productive conversations with enterprise AI stakeholders.

These templates have been refined across multiple enterprise AI engagements. They are opinionated about what matters and what does not.

---

## How to Use These Templates

**Tier 1 — Diagnostic Assessment:**
Use the Use Case Intake template to document AI use cases discovered during the assessment. Use the Prioritisation Matrix to produce the ranked use case portfolio that forms the primary output of a Tier 1 engagement.

**Tier 2 — Architecture Blueprint:**
Use the Business Case template to justify the investment in the highest-priority use cases. Use the Target Architecture template to document the recommended technical architecture for each use case. Use the Roadmap template to structure the phased delivery plan.

**Tier 3 — Implementation Advisory:**
Use all templates as living documents, updated as the programme progresses. The Use Case Intake and Business Case templates become the source of truth for the use case portfolio. The Target Architecture and Roadmap are updated at each milestone.

---

## Template Conventions

**Bracket notation:** Text in `[brackets]` is a prompt or placeholder to be filled in with client-specific information.

**Inline guidance:** Some sections include guidance text in italics explaining the intent of the section. Remove the guidance text from the final client deliverable.

**Ratings:** Where ratings (1–5, High/Medium/Low) appear, the rating criteria are defined within the template. Apply them consistently across all use cases in a portfolio to ensure comparable prioritisation.
